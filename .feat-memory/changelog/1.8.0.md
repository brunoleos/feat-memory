# 1.8.0 — 2026-06-26

`release` ganha automação git; a política de tags muda.

### Mudado

**`feat-memory release` não bumpa mais VERSION; tagueia a versão corrente (ADR-0045).** O bump de VERSION continua per-commit (ADR-0020 — a versão evolui visível), mas a criação de **git tag passa a ocorrer só no `release`**, com o nome da versão atual (`v<VERSION>`). O comando congela o UNRELEASED em `changelog/<VERSION>.md`, commita e cria a tag; flags `--no-commit`/`--no-tag` para revisão. Elimina a poluição de uma tag por commit.
